# 项目协作约束

每次提交、推送或创建 PR 前，必须先读取 `docs/pr-guidelines.md`，并按其中最新约束执行。

每次修改代码、运行服务、排查问题或提交 PR 前，必须先读取 `docs/chinese-logging-comments-guidelines.md`，并严格遵守中文日志与中文注释约束。

当前关键约束：

- 提交信息使用中文 Conventional Commits，例如 `feat: 新增异步叙事分析流程` 或 `fix: 修复本地环境变量加载顺序`。
- 推送说明、提交总结、PR 描述、后端应用日志和关键运行日志使用中文。
- 所有新增业务日志、调试日志、异常日志、前端用户提示和代码注释必须使用中文。
- 不得在日志中打印完整 API Key 或完整小说正文。
- 真实 `.env`、缓存、日志、构建产物、依赖目录和本地测试文本不得提交。
