# 页面功能介绍文字稿

下面是一版偏口语化的页面讲解稿，适合录屏演示、项目汇报或产品介绍时使用。

大家现在看到的是“小说转剧本”的创作工作台。它的目标不是简单把小说改写成一段文本，而是把小说改编拆成一条更接近影视开发的流程：先导入小说，再分析人物、事件和场景，然后生成剧本、分镜图片，最后继续推进到视频生成任务。

首先是“小说导入”页。这里可以上传 TXT、Markdown 或 Docx 文件。上传后系统会先做自动分章，我们可以马上看到章节列表。接着点击“开始叙事分析”，后端会调用 DeepSeek，把每一章里的角色、事件、地点、时间、冲突、对话、环境和分镜规划都抽出来。页面左侧还有本地小说库，方便在多个小说项目之间切换。

分析完成后，可以进入“角色管理”。这里每个角色都会生成一张人物卡，包括角色描述、别名、重要性、出场章节，还有对应的原文依据。它比较适合用来快速检查人物是不是漏了、别名有没有混在一起、主次角色是否合理。

接下来是“人物关系图”。这个页面会把人物变成节点，把人物之间的关系变成连线。关系越强，视觉上就越明显。我们点某个人物，还可以跳回角色卡片继续看详情。这个页面适合从整体上看小说里的人物网络。

“章节/事件时间线”是按章节来浏览剧情。每一章下面会展示抽取出的关键事件，包括事件摘要、地点、时间、冲突和结果。点击章节或事件卡片，可以打开原文比对，直接看到模型判断是从哪一段小说里来的。

然后是“场景拆分板”。它会先把连续章节整理成总场景，再拆成更细的子场景。每个子场景都会关联事件、人物、环境、对话和分镜规划。这个页面相当于从小说叙事过渡到影视场景的中间层，是后面生成剧本和分镜的基础。

进入“剧本生成”页之后，左侧是场景列表，右侧是当前场景的剧本编辑区。页面会把场景里的人物、事件、环境、分镜和对话参考都放在编辑器旁边。我们可以自己写，也可以点击 AI 自动补全，让 DeepSeek 基于这些结构化素材生成单场剧本。生成后还可以继续人工修改，并保存到本地草稿。

“剧本总览”页用来统一查看已经保存的场景剧本。这里能看到完成了多少场，每场有多少文字，也可以一键导出 YAML。这个 YAML 后续可以给其他工具、流水线或审阅流程继续使用。

再往后是“分镜生图”。这个页面会读取剧本里的场景和镜头规划。我们可以选择完整镜头，也可以选择构图、焦点、视角、情绪、转场这些单个定帧。系统可以先用 DeepSeek 生成图片提示词，再调用 Seedream 生成分镜参考图。这里生成的图片主要是为后续视频生成提供首帧或视觉参考。

“分镜管理”页负责管理这些生成出来的图片任务。我们可以看图片预览、任务状态、本地保存路径，也可以通过小说、场景、分镜标签跳回对应页面。删除的任务会先进入回收箱，避免误删。

“视频生成”页就是把前面的剧本和分镜图片继续往视频任务推进。这里既可以导入完整剧本，也可以只导入某一个场景、某一个镜头，甚至某一个小分镜粒度。页面支持选择分镜图片作为首帧或参考图，也可以上传图片、视频、音频素材，然后配置 Seedance 的模型、画幅、时长、清晰度和 Seed，最后创建视频生成任务。

最后是“视频管理”。这里会展示所有视频任务，包含任务状态、生成结果、剧本预览、素材数量和本地视频路径。页面会自动同步未完成的 Seedance 任务，也支持手动同步。每个任务都有小说、章节、场景、镜头、分镜图片这些标签，所以生成完之后还能很快追溯它是从哪个文本和哪个场景来的。

整体来看，这个项目的核心价值是把小说改编过程拆成可检查、可编辑、可追溯的多个步骤。模型负责做初步抽取和生成，人负责校对、筛选和继续创作。它不是一次性给出最终答案，而是提供一个从小说文本走向剧本、分镜和视频素材的创作工作流。
